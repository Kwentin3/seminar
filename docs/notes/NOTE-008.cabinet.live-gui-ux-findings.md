# Cabinet Live GUI UX Findings

## Purpose
Capture the narrowest useful follow-up actions after the first real production GUI walkthrough as admin and lecturer.

## High-Value Follow-Ups
1. Reduce card density in the library list:
   - keep status, recommendation, reading mode, summary, and theme prominent;
   - demote source path and some tags from the first scanning layer.
2. Make the core lecture-prep subset feel more obviously “trusted first” without inventing a new workflow.
3. Shorten some internal-tool phrasing so the cabinet reads less like a registry and more like a preparation workspace.
4. Improve narrow-screen reader ergonomics by letting the article body start sooner after the metadata block.

## What Not To Turn This Into
1. Do not expand this into a full redesign.
2. Do not introduce CMS/editor workflow under the guise of UX cleanup.
3. Do not widen the role model just because admin and lecturer currently look similar.

## Why This Matters
1. The cabinet is already useful.
2. The next gains come from reducing friction and visual noise, not from adding new product surface.
